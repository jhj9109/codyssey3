# 🧠 Mini NPU Simulator

이 프로젝트는 AI 하드웨어(NPU)의 핵심 연산인 MAC(Multiply-Accumulate)을 소프트웨어로 직접 구현하고, 데이터 크기 증가에 따른 연산 병목 현상과 NPU의 병렬 처리 필요성을 체감하기 위한 시뮬레이터입니다.

## 🚀 실행 방법

프로젝트 최상단 디렉토리에서 아래 명령어를 통해 시뮬레이터를 실행할 수 있습니다.

```bash
python main.py
```

- 모드 1: 사용자로부터 3x3 필터 2개와 패턴을 직접 입력받아 판정합니다.
- 모드 2: data.json 파일을 자동으로 로드하여 다수의 패턴을 평가하고 전체 결과 리포트를 출력합니다.
- 테스트 실행: python -m tests.test_core 명령어로 순수 함수들에 대한 단위 테스트를 수행할 수 있습니다.

## 💡 핵심 구현 요약

1. 라벨 정규화

- (Label Normalization)입력 데이터의 다양성(+, x, cross, Cross 등)으로 인한 로직 복잡도를 줄이기 위해, 프로그램 진입 단계에서 모든 라벨을 Cross와 X 두 가지 표준 라벨로 변환하여 내부 처리를 일원화했습니다.

2. MAC 연산 구현

- 외부 벡터화 라이브러리(NumPy 등)를 배제하고, 순수 파이썬의 중첩 반복문(for loop)을 사용하여 입력 배열과 필터 배열의 같은 위치 값을 곱하고(Multiply) 누적하는(Accumulate) NPU의 하드웨어적 연산 흐름을 모사했습니다.

3. Epsilon 기반 점수 비교

- 실수(Float) 데이터를 다룰 때 발생하는 미세한 부동소수점 오차로 인한 오작동을 방지하기 위해, 두 점수의 차이가 1e-9 미만일 경우 동점(UNDECIDED)으로 간주하는 안전장치를 적용했습니다.

## 📊 결과 리포트 (Fail 원인 분석)

data.json 기반의 모드 2 자동 평가 결과, 고의로 삽입된 실패 케이스를 프로그램 중단 없이 정확히 식별해 냈습니다.

- 발견된 FAIL 케이스: [size_5_02_intentional_fail]
- 원인 분석: 해당 데이터는 입력 배열 패턴이 명백한 'X' 형태(양 대각선이 1인 배열)를 띄고 있어 MAC 연산 결과 또한 'X' 필터에서 압도적으로 높은 점수가 산출되었습니다. 하지만 JSON 파일 내의 기대값(expected)이 'cross'로 잘못 기재되어 있어, 실제 연산 판정(X)과 기대값(Cross)의 불일치로 인해 정상적으로 FAIL 처리되었습니다.
- 결론: 그 외 노이즈가 섞인 실수형 데이터나 라벨이 제각각인 데이터들은 모두 PASS 하였습니다. 이는 Epsilon 정책과 라벨 정규화가 의도한 대로 완벽하게 작동하여 불필요한 연산/로직 실패(0건)를 방어해 냈음을 의미합니다.

### ⏱️ 성능 분석 (시간 복잡도 증명)

시뮬레이터를 통해 다양한 크기(N×N)의 행렬에 대해 MAC 연산을 10회 반복 측정하여 I/O가 배제된 순수 연산 평균 시간을 측정했습니다.

- 3x3 (연산 9회) : 약 0.001 ms 내외
- 5x5 (연산 25회) : 약 0.002 ms 내외
- 13x13 (연산 169회) : 약 0.015 ms 내외
- 25x25 (연산 625회) : 약 0.060 ms 내외

### [O(N²) 시간 복잡도 분석]

위 성능 분석 표의 결과를 보면 데이터의 크기(N)가 증가함에 따라 연산 시간이 급격하게 상승하는 것을 볼 수 있습니다. MAC 연산은 2차원 배열을 순회하기 위해 필수적으로 중첩 반복문(for i, for j)을 사용하므로, N이 커질수록 연산 횟수는 N의 제곱인 $N^2$에 비례하여 증가합니다. 즉, 시간 복잡도는 $O(N^2)$를 가집니다.
현대의 고도화된 AI 모델은 25x25를 넘어 수백, 수천 크기의 필터를 수십억 번 적용해야 합니다. 소프트웨어(CPU) 기반의 직렬 처리로는 이러한 $O(N^2)$의 연산 폭발을 감당하기 어렵기 때문에, 수천 개의 단순 MAC 연산을 동시에 병렬로 처리할 수 있는 NPU(Neural Processing Unit) 하드웨어의 도입이 필연적이라는 것을 이번 성능 측정 결과를 통해 수학적, 경험적으로 증명할 수 있었습니다.
